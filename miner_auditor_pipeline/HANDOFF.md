# Miner -> Auditor Local Pipeline Handoff

## Scope

This folder contains a standalone local pipeline for:

1. running `task1.2 / miner / extract` with local `unsloth + QLoRA adapter` inference
2. converting miner outputs into an auditor-ready HF dataset
3. running `task1.3 / auditor` inference through the existing `sglang + qbatch.py` path
4. running GRPO eval through `sglang` on local `hf_cot_sft`, with optional TTC

This folder is intentionally limited to the `miner -> auditor` chain only.
The folder now also keeps local dataset copies under `data/`, while model weights remain outside this folder.

## Files

- `common.py`
  Shared helpers for path resolution, extraction parsing, factor aggregation, and auditor dataset row construction.
- `data/`
  Local dataset copies used by this folder.
- `run_miner_pipeline.py`
  Step 1. Local miner inference. Produces both JSON outputs and an auditor-consumable HF dataset.
- `run_auditor_pipeline.py`
  Step 2. Auditor inference wrapper. Calls the existing `qbatch.py --mode auditor`.
- `run_grpo_eval.py`
  Standalone GRPO eval on local `hf_cot_sft` through `sglang`, with optional TTC.
- `run_miner_pipeline.sh`
  Shell wrapper for step 1.
- `run_auditor_pipeline.sh`
  Shell wrapper for step 2.
- `run_grpo_eval.sh`
  Shell wrapper for GRPO eval.

## Design Choices

- Miner inference uses `base model + adapter` only.
  It does not depend on a merged miner model directory.
- Miner prompt formatting reuses the existing factor-level logic from `build_sft_extract_v5.py`.
- Auditor prompt formatting reuses the existing prompt builder from `task13_dataset_common.py`.
- Auditor inference is not reimplemented here.
  Step 2 delegates to the repository's existing `qbatch.py` auditor-only path.
- The implementation prefers the most stable local path over a more optimized but riskier path.

## Miner Step

### What it does

`run_miner_pipeline.py`:

1. selects filings from either:
   - raw MDA files
   - or an existing factor-level miner HF dataset
2. supports:
   - split selection: `train | validation | test`
   - limiting number of tickers
   - limiting number of filings
   - choosing all factors vs only teacher-answered factors
3. runs factor-level miner inference locally with `unsloth`
4. writes factor-level and filing-level miner JSON outputs
5. builds a new HF dataset that can be passed directly to `qbatch.py --mode auditor`

### Important arguments

- `--input-source {factor_dataset,raw}`
- `--factor-dataset`
- `--raw-root`
- `--extract-root`
- `--split {train,validation,test}`
- `--question-mode {all,teacher_answered}`
- `--max-tickers`
- `--max-filings`
- `--output-dir`

`--question-mode` semantics:

- `all`
  Run all factors found in the global factor/question schema.
- `teacher_answered`
  Keep only the factors that have at least one teacher-answered extraction for this filing, but still feed the full candidate question list for each kept factor.

### Output artifacts

Inside the miner output directory:

- `selected_filings.json`
  Which filings were selected.
- `miner_pred_rows.json`
  Factor-level miner predictions.
- `miner_filing_aggregates.json`
  Filing-level aggregated miner outputs.
- `auditor_rows_preview.json`
  Lightweight preview of the generated auditor rows.
- `summary.json`
  High-level miner step summary.
- `auditor_hf_dataset/`
  The key artifact for step 2.

## Auditor Step

### What it does

`run_auditor_pipeline.py`:

1. takes the HF dataset generated by the miner step
2. calls the repository's existing `qbatch.py --mode auditor`
3. writes auditor prediction CSV and JSON outputs

### Start `sglang` separately first

This pipeline assumes the `sglang` server is started in a separate shell or tmux pane before step 2.

Current GRPO serving model example:

`/scratch/xla2767/hold2/models/qwen3_8b_thinking_grpo_merged_v1`

Example command:

```bash
python -m sglang.launch_server \
  --model-path /scratch/xla2767/hold2/models/qwen3_8b_thinking_grpo_merged_v1 \
  --port 30000 \
  --dtype bfloat16 \
  --reasoning-parser qwen3 \
  --mem-fraction-static 0.85
```

If the serving path changes later, replace only `--model-path`.

### Important arguments

- `--auditor-dataset`
- `--model-path`
- `--sglang-url`
- `--split {train,validation,test}`
- `--batch-size`
- `--max-batches`
- `--output-dir`

### Output artifacts

Inside the auditor output directory:

- `auditor_predictions.csv`
- `auditor_predictions.json`
- `summary.json`

## Dataset Contract for Step 2

The generated `auditor_hf_dataset` is designed to match the style expected by current auditor tooling:

- it contains `messages`
- `messages[1]["content"]` is the current auditor user prompt
- it includes the usual metadata fields such as:
  - `ticker`
  - `form`
  - `report_date`
  - `factor`
  - `evidence_count`
  - `historical_count`

This is sufficient for `qbatch.py --mode auditor`, which currently reads the auditor prompt from `messages[1]["content"]`.

## Notes and Constraints

- Miner is intentionally local `unsloth + adapter`.
  This follows the current stability assumption in the repo.
- Auditor is intentionally left on `sglang`.
- GRPO eval in this folder reads the local copied `data/hf_cot_sft` by default.
- The miner-generated auditor dataset may include teacher metadata when a matching `TASK13_AUDIT` file exists, but step 2 does not require that teacher target at inference time.
- If miner returns no evidence for a factor, that factor is dropped by default unless `--keep-empty-factors` is used.
- The default implementation is stable-first, not speed-first.

## GRPO Eval

### What it does

`run_grpo_eval.py` runs GRPO eval against a separately started `sglang` server using local `hf_cot_sft`.

### SGLang + TTC behavior

- `--ttc-n 1`
  No TTC. One completion per example.
- `--ttc-n >1`
  TTC enabled. The script samples multiple completions per example and aggregates them by label vote.
- `--enable-thinking` / `--disable-thinking`
  Controls whether the request enables Qwen thinking mode.

### Minimal command

```bash
python /projects/p32908/nlp_code/miner_auditor_pipeline/run_grpo_eval.py \
  --sglang-url http://localhost:30000 \
  --ttc-n 1 \
  --split test \
  --max-filings 10
```

### Default data and model behavior

- dataset:
  `data/hf_cot_sft`
- model weights:
  served externally by the running `sglang` server
- outputs:
  `/gpfs/projects/p32908/nlp_result/miner_auditor_pipeline/grpo_eval`
